REST API для роботи з коллекцією оголошень

  • Для додавання нового оголошення необхідно відправити POST - запрос на ендпоінт: /create-ad

                    в body {
                    "title": "car",
                    "description": "it is great car",
                    "price": 5900,
                    "links": ["https://www.images-holder/2123.jpg", "https://www.images-holder/23.jpg", "https://www.images-holder/123.jpg"]
                }

    У відповіді на запит прийде ID створеного оголошення

  • Для отримання оголошення необхідно відправити GET - запрос на ендпоінт: /ads/:id/

                    Назву опціональних полів необхідно передати у параметр fields 
                припустимі значення: 'description', 'links'
                приклад: GET  /ads/3432323232344?fields=description,links
                або: GET  /ads/3432323232344?fields=links

  • Для отримання списку оголошеннь необхідно відправити GET - запрос на ендпоінт: /ads

                    Опціональні поля:
                    sort_by_price, sort_by_date
                    припустимі значення: 1, -1

                    title
                    припустимі значення: фрагмент тексту по якому буде проводитись пошук у назві оголошення

                    page 
                    припустимі значення: номер сторінки

                приклад: GET  /ads?title=car&sort_by_price=-1

Список ендпоінтів:

  • POST /create-ad
  • GET: /ads/:id/
  • GET: /ads
